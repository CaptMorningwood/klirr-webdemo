# PAUSED / FUTURE EXPLORATION: Premium

## 1. Decision

Premium is paused as an active product. Klirr currently has one product level.

## 2. Reason

The current Premium experiments do not yet provide a sufficiently clear, mature, recurring value to justify a separate tier.

## 3. Preserved work

The following work is preserved for future exploration and backward compatibility:

- Entitlement foundation
- Value analysis
- Goals
- History
- Monitoring
- Workspaces
- Shared Budget demo
- Versions
- Buddy capability model
- Privacy integration

## 4. Active product

The active Klirr product has:

- One product level
- Full Budget core
- Full Budget Buddy
- Optional AI that requires explicit, versioned consent

AI consent controls external AI calls. It does not control access to Budget Buddy.

## 5. Evidence required before reintroduction

Premium may be reconsidered only when there is evidence such as:

- Repeated user demand
- Clear recurring value
- High engagement with one coherent feature set
- Meaningful retention benefit
- Feature maturity
- Production-ready backend
- Willingness-to-pay validation
- Simple positioning

## 6. Reactivation checklist

Before Premium returns, Klirr must:

- Choose one Premium outcome
- User test it
- Define pricing
- Add security and server enforcement
- Add billing
- Restore entitlement gates intentionally
- Complete data handling
- Complete App Store disclosure
- Run QA
