# Klirr roadmap inför betalning och App Store

Detta återstår innan Klirr bör lanseras med riktig betalversion och mobil distribution:

- Stabil Supabase persistence med tydlig konflikt- och offlinehantering.
- Riktig user profile som äger hushållsprofil, abonnemangsstatus och preferenser.
- Subscription backend som validerar kvitton/checkout och synkar entitlements.
- Stripe för webbbetalningar.
- StoreKit/In-App Purchase för iOS.
- TestFlight-flöde med intern och extern testgrupp.
- App Store metadata: namn, kategori, screenshots, support-URL och granskningsanteckningar.
- Integritetspolicy som täcker lokal lagring, Supabase och AI-användning.
- Användarvillkor för budgetverktyg och ansvarsbegränsningar.
- Demo-konto och granskningsinstruktioner för Apple review.
- Robust export/radera användardata även för molndata.
- Onboarding för manuell start, CSV-import och hushållsprofil.
